# 登入資訊

## 正確的登入方式

根據 seed 文件，正確的登入資訊如下：

### 系統管理員
- **員工編號**: `ADMIN001` (大寫) 或 `admin001` (小寫，已支援)
- **密碼**: `admin123`
- **權限**: 系統管理員 + 部門主管

### 一般員工
- **員工編號**: `EMP001` (大寫) 或 `emp001` (小寫，已支援)
- **密碼**: `user123`
- **權限**: 一般員工

## 重要提醒

1. **員工編號現在支援大小寫不敏感**：可以使用 `ADMIN001`、`admin001` 或 `Admin001` 登入
2. **密碼區分大小寫**：密碼必須完全匹配
3. **如果登入失敗，請檢查**：
   - 是否已運行資料庫遷移：`npm run migrate`
   - 是否已運行種子資料：`npm run seed`
   - 員工編號是否正確（可以是任意大小寫）
   - 密碼是否完全匹配（區分大小寫）

## 故障排除

如果仍然無法登入：
1. 檢查後端服務器控制台日誌（會顯示登入嘗試和錯誤）
2. 確認資料庫中是否有用戶資料
3. 確認用戶的 `is_active` 欄位是否為 `true`
